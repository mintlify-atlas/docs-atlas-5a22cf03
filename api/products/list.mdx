---
title: "List Products"
api: "GET /products"
---

Retrieve a paginated list of products with optional filtering by category.

## Authentication

This endpoint is public and does not require authentication.

## Query Parameters

<ParamField query="category_id" type="number">
  Filter products by category ID
</ParamField>

<ParamField query="take" type="number" default="10">
  Number of products to retrieve per page
</ParamField>

<ParamField query="skip" type="number" default="0">
  Number of products to skip (for pagination)
</ParamField>

## Response

<ResponseField name="products" type="array">
  Array of product objects
  
  <Expandable title="Product Object">
    <ResponseField name="id" type="integer">
      The unique identifier for the product
    </ResponseField>
    
    <ResponseField name="name" type="string">
      The product name
    </ResponseField>
    
    <ResponseField name="price" type="number">
      The product price
    </ResponseField>
    
    <ResponseField name="image" type="string">
      The product image URL or filename
    </ResponseField>
    
    <ResponseField name="inventory" type="number">
      The quantity in stock
    </ResponseField>
    
    <ResponseField name="categoryId" type="integer">
      The category ID
    </ResponseField>
    
    <ResponseField name="category" type="object">
      The category object with full details
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of products matching the query
</ResponseField>

## Pagination

The products are returned in descending order by ID (newest first). Use the `take` and `skip` parameters for pagination:

- **Page 1**: `take=10&skip=0`
- **Page 2**: `take=10&skip=10`
- **Page 3**: `take=10&skip=20`

The `total` field in the response indicates the total number of products matching your query.
